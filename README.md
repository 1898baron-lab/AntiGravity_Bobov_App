# AntiGravity Local RAG Agent

## О проекте
Это локальная система RAG (Retrieval-Augmented Generation) для работы с вашими личными знаниями. Проект строится так, чтобы разделить:

* **Инструменты (`.docs/`)** — документация сторонних библиотек и вспомогательные материалы
* **База знаний (`knowledge_base/`)** — ваши заметки, проекты и контент, на который агент опирается
* **Код агента (`agent/`)** — логика поиска, фильтрации, вызовов локальной модели

Проект работает автономно и конфиденциально: текстовые данные остаются на вашем компьютере.

## Цель
Создать понятную, локальную платформу для персонального AI-ассистента, который отвечает на вопросы по вашей голове знаний и не путает её с документацией инструментов.

## Структура проекта
```text
.
├── .docs/                 # Сторонняя документация, README библиотек, справочные материалы
├── agent/                 # Код локального агента
│   ├── main.py
│   ├── config.py
│   └── utils/
├── knowledge_base/        # Ваши заметки и проекты для индексации агентом
│   ├── notes/
│   ├── projects/
│   └── archive/
├── requirements.txt       # Python-зависимости
├── .env.example           # Пример конфига с ключами
└── README.md              # Главный манифест проекта
```

## Как начать
1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
2. Скопируйте `.env.example` в `.env` и заполните нужные переменные.
3. Заполните `knowledge_base/` вашими заметками и проектными документами.
4. Запустите агента:
   ```bash
   python agent/main.py --query "Напиши краткий план по текущему проекту"
   ```

## Важные правила
* Папка `.docs/` не должна индексироваться агентом как база знаний.
* `knowledge_base/` содержит только тот контент, который вы хотите передавать агенту.
* Код агента живёт в `agent/`.

## Рекомендация для агента
В конфиге агента уже задан список исключений:
```python
ignore_dirs = {".git", ".docs", "venv", ".venv", "__pycache__", "node_modules"}
```
Это поможет избежать путаницы между вашей реальной задачей и документацией сторонних библиотек.

## Что дальше
* Перенесите сторонние README и справочные файлы в `.docs/`
* Добавьте свои заметки и проекты в `knowledge_base/`
* Используйте `agent/main.py` для запуска локального RAG
