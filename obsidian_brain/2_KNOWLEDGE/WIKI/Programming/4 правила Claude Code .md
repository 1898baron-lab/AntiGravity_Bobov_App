> [!abstract] ИИ Выжимка
> Статья представляет четыре ключевых правила для максимальной эффективности при использовании ИИ в кодинге, особенно с Claude. Основные принципы включают требование к модели задавать уточняющие вопросы вместо догадок, написание минимально необходимого кода без избыточности, строгое следование запросу и описание желаемого результата, а не предоставление пошаговой инструкции. Автор также описывает практический способ внедрения этих правил в рабочий процесс проекта через специальный файл-набор инструкций.

4 правила Claude Code 
Как использовать ИИ-кодинг на максимум и избежать типичных ошибок
Правило №1: не угадывай — спрашивай
Патология всех LLM. LLM-модели всегда угадывают. Например, ты пишешь «Добавь валидацию 
email» — Claude выбирает один из 10 способов, но не объясняет почему.
90%+ ошибок в LLM возникают потому, что модель угадывает вместо того, чтобы 
переспросить
Правило: Claude должен явно задавать уточняющие вопросы перед тем, как принять решение 
самостоятельно
Правило №2: не пиши лишнего
Раздутый код = потраченные токены. Claude обучался на продакшн-версиях кода, которые чаще 
всего раздуты. Просишь функцию на 10 строк — получаешь 200.
Проблема
Лишний код сжигает токены, усложняет 
проект и создаёт новые баги
Тест Карпати
Что сказал бы старший инженер? Если «это 
переусложнено» — переписать
Правило: Писать минимально необходимый код. Никакой избыточности.
Правило №3: делай только то, о чём просят
Самая коварная проблема. Попросил поменять иконку — Claude поменял иконку и добавил 5 страниц. Код не слушает задачу целиком и делает десяток лишних действий: что-то удаляет, что-то меняет.
Правило: Claude должен делать строго то, о чём его попросили — и ничего другого
Правило №4: описывай результат, а не инструкцию
❌ Слабый запрос
Почини баг с сортировкой
→ Claude начинает спрашивать: какой алгоритм? направление?
✅ Сильный запрос
Не «почини баг», а «напиши тест, который воспроизводит баг, потом сделай, чтобы он прошёл»
→ Claude сам ищет правильное решение
Не говори КАК — говори ЧТО должно получиться. Пусть Claude сам найдёт путь к нужному результату.
Как внедрить скилл в свой проект
Найди файл Claude.md в своём проекте
(он есть у каждого, кто вайбкодит — даже если не создавал его вручную)
Скачай скилл с GitHub
https://github.com/forrestchang/andrej-karpathy-skills 
Открой Claude и напиши
"Скачай данный Claude.md, изучи наш Claude.md и добавь правила, которые будут полезны нашему проекту"
Claude сам интегрирует правила
как дополнение к существующим — не заменяя их
Скилл работает с Claude Code, Cursor, ChatGPT Codex и другими инструментами
Как установить скилл — два способа
Способ 1 — в отдельный проект (5 секунд)
curl -o CLAUDE.md 
https://raw.githubusercontent.com/forrestch
ang/andrej-karpathy-skills/main/CLAUDE.md
Кладёшь файл в корень проекта — Claude 
Code читает его автоматически.
Способ 2 — глобально на всю систему
Ставишь этот же файл в домашнюю 
директорию (~/.claude/) — и правила 
применяются ко всем проектам сразу.
Бонус для Cursor: в репозитории есть версия для Cursor (.cursor/rules/karpathy-
guidelines.mdc). Кто работает в Cursor — копируешь туда.
