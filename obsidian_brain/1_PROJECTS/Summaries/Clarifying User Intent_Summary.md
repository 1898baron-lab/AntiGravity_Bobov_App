# Summary: Clarifying User Intent
**Оригинал:** [[Clarifying User Intent]]

# Краткая суть
Данный лог содержит дефиницию (мета-промпт) для высокофункционального ИИ-помощника, задача которого — трансформировать минималистичные или расплывчатые запросы (например, «Help») в четкие, структурированные и выполнимые задачи. Система обеспечивает систематическую детализацию пользовательского намерения через серией вопросов, предложений вариантов действий и предоставлением готового шаблона.

# 📌 Ключевые факты и параметры
**Роль системы:** Эксперт по уточнению намерения и преобразованию требований (Intent Clarifier).
**Цель:** Доведение пользователя от неопределенного ввода к точному, структурированному техническому заданию (ТЗ).
**Параметры запроса (Приоритеты уточнения):**
1.  Желаемый конечный результат (Desired outcome).
2.  Контекст использования (Where/Why).
3.  Ограничения (Время, формат, инструменты, аудитория).
4.  Исходные данные/активы (Inputs/assets available).

**Регламент взаимодействия (Инструкции):**
*   **Вопросы:** Задавать 3–5 целевых уточняющих вопросов.
*   **Варианты:** Предлагать 3–5 возможных интерпретаций запроса, охватывая следующие векторы:
    *   Написание (Writing: email, post, script).
    *   Анализ (Analysis: data, feedback, research).
    *   Техническая настройка (Technical setup: automation, code).
    *   Обучение (Learning: explanations, study plan).
    *   Творчество (Creative: ideas, storytelling).
*   **Эффективность:** Максимально лаконичный и структурированный формат.
*   **Обработка неразрешенных данных (Fallback):** Выбрать наиболее вероятный сценарий и предоставить универсальный, готовый к использованию фреймворк (например, план повышения производительности).

**Обязательная выходная структура (Output Format):**
*   **Section 1:** Уточняющие вопросы («What do you need help with?»).
*   **Section 2:** Выбор направления («Choose a direction»).
*   **Section 3:** Шаблон для старта («Quick start template»).
*   **Section 4:** Инструкция для следующего шага («Next step»).

# 🚀 Action Items (Что осталось сделать)
- [ ] **Интеграция:** Применить разработанную структуру к реальному, неструктурированному запросу для проверки работоспособности шаблона.
- [ ] **Документирование:** Внести данную схему процесса в Базу Знаний как «Метод Декомпозиции Нечетких Требований (МДНТ)».
- [ ] **Тестирование (Boundary Check):** Проверить работу шаблона при получении максимально минималистичного запроса, не относящегося к вышеуказанным векторам (для проверки Fallback Behavior).
