Методичка «Проекты (Project Workspaces) в ChatGPT»

Цель — дать инженеру ПТО пошаговую, технически-точную инструкцию, как использовать Projects для долгосрочной работы с ППР, РД и шаблонами, минимизируя потерю контекста и путаницу при смене моделей.

1. Что такое Project

Project — изолированное рабочее пространство, в котором объединяются:

чаты;

файлы (до 20 шт. каждый ≤ 100 МБ);

индивидуальные инструкции («Project instructions»).

Контекст каждого чата внутри проекта «подтягивает» файлы и инструкции проекта, улучшая связность ответов 
OpenAI Аида
OpenAI Аида
.

2. Создание и первичная настройка
Шаг	Действие	Комментарий
1	Нажмите ⋯ → Move to project (или «New project»)	Создаётся пустой workspace.
2	Задайте Project Goal	Коротко: «Разработка ППР HVAC для АЭС ‘Аккую’».
3	Напишите Project Instructions	Укажите правила работы (шаблоны, ГОСТ 21.110-2013, приоритет HV / TM и т.д.).
4	Перетащите существующие чаты или начните новый	Для миграции историй обсуждений.
5	Загрузите файлы в структурированные папки	См. реком. структуру § 4.
3. Работа с файлами

Любой загруженный файл доступен во всех чатах проекта.

Файл остаётся на сервере пока жив сам проект 
OpenAI Аида
.

Рекомендация: именуйте файлы по схеме 2025-07-11_RD_HV_00USV.pdf — это облегчает поиск.

4. Рекомендуемая структура папок для ППР-инженера
/01_RD          (HV, TM, KM, TZ, AR)
/02_PPR         (Шаблоны, версии, графика)
/03_TI_KMD      (технологические инструкции, КМД)
/04_Reviews     (замечания Заказчика, Excel-трекеры)
/05_Analytics   (выжимки, отчёты, расчёты)
/ZZ_Archive     (закрытые разделы)

Храним только «живые» данные; старые версии переносим в /ZZ_Archive — уменьшает шум при поиске.

5. Управление моделями внутри проекта
Задача	Оптимальная модель	Причина
Быстрый черновик, обсуждение графики	GPT-4o	Реагирует на PDF + DWG, быстрее формулирует идеи.
Глубокая инженерная проверка, расчёты	o3 / o4-mini	Сильнее в reasoning и тех-анализе.
Итоговая верстка текстов ППР	GPT-4.1 / GPT-4.1 mini	Лучшее следование инструкциям, аккуратный русский.

Важно: контекст проекта не «сливается» между моделями. При смене модели:

сделайте краткое резюме («Что уже решено…»);

явно сослитесь на нужные файлы.
Это снижает риск расхождения ответов 
OpenAI Аида
OpenAI Аида
.

6. Лайф-хаки для непрерывности контекста

Чат-резюме: держите в фиксированном сообщении-шпоре краткую сводку ключевых решений; обновляйте при каждом значимом шаге.

Версия-контроль: после каждой крупной правки ППР добавляйте файл вида v02_2025-07-11_PPR.docx.

Метод «One Source»: при анализе РД цитируйте конкретные страницы; другой чат в том же проекте сможет использовать точную ссылку.

Цветовые маркеры: в названиях чатов добавляйте ⚙️ (тех.обсуждение), 📝 (редактура), 📊 (аналитика).

7. Ограничения и подводные камни
Ограничение	Митигирующее действие
20 файлов / проект	Разбивайте на проекты по объектам (например, 00USV, 00UST).
Разная «память» моделей	Не меняйте модель без необходимости; фиксируйте резюме.
Нет автоматического обновления файла	При загрузке новой версии используйте новое имя; уточняйте модели, какой файл актуален.
Размер контекста ≤ 128 к токенов (GPT-4o)	Длинные ПЗ дробите на секции, подгружайте по частям.
8. Безопасность и конфиденциальность

Проекты не шарятся вне вашей учётки; при удалении проекта все файлы стираются через ≤ 30 дней 
OpenAI Аида
.

Team/Enterprise-аккаунты дополнительно получают шифрование, аудит-логи 
Tom's Guide
.

Не храните в Projects персональные данные работников без анонимизации.

9. Быстрый чек-лист (повесь над монитором)

Сразу загрузил только актуальные РД/ППР.

Каждую сессию заканчиваю двумя строками резюме.

Новая версия файла → новое имя + обновить резюме.

Перед сменой модели — дай модели резюме и список ключевых файлов.

10. Итог

Используя Projects как «виртуальную папку» для ППР, ты избавляешься от рассыпанных файлов и несогласованных версий. Главное — дисциплина: единая структура, фиксирование решений и осторожная смена моделей. При таком подходе ChatGPT становится полноценным онлайн-«ПТО-ассистентом», который «помнит» твой объект и не задаёт лишних вопросов.
